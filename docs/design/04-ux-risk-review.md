# UX Risk Review

## High-risk UX areas
- false confidence after upload
- unclear delete semantics
- hidden dependency on a disconnected disk
- confusing primary vs replica roles
- online preview mistaken for full archival safety

## Mandatory mitigations
- distinct status chips and copy
- cleanup screen separate from archive screen
- device/storage health visible from overview
- per-asset detail shows placements and verification state
- every destructive path explains remaining copies
