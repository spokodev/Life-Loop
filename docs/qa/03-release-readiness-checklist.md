# Release Readiness Checklist

- product docs updated
- ADR needed? if yes, written
- schema changes reviewed
- backup/restore impact assessed
- new permissions reviewed
- smoke tests pass
- destructive actions manually exercised in non-prod
- rollback notes documented
- positive, negative, partial-success, and dependency-loss states manually reviewed
- reduced-motion mode manually reviewed on animated screens
- critical screens remain understandable with animation disabled
